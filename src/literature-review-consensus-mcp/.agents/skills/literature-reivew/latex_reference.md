# LaTeX Reference For Literature Reviews

This document is separate from the skill file and explains the LaTeX conventions used for literature reviews in this skill.

## Skill files

- `SKILL.md`: skill instructions
- `latex_reference.md`: LaTeX usage guidance
- `render_latex_pdf.py`: Python helper that compiles the generated `.tex` file

## Generated output files

- `literature_review.tex`: the generated LaTeX source in the current working directory
- `literature_review.pdf`: the rendered PDF when compilation succeeds

## Minimal document structure

Use a self-contained `article` document:

```tex
\documentclass[11pt]{article}
\usepackage[margin=1in]{geometry}
\usepackage[T1]{fontenc}
\usepackage[utf8]{inputenc}
\usepackage{graphicx}
\usepackage{hyperref}
\usepackage{xurl}
\hypersetup{hidelinks,breaklinks=true}

\title{Literature Review Title}
\author{Generated by Codex}
\date{\today}

\begin{document}
\maketitle

\section{Question}
...

\section{Overview}
...

\section{Main Findings}
...

% Optional figures
\input{literature_review_images.tex}

\section{Gaps or Limitations}
...

\section{Bottom Line}
...

\section{References}
\begin{enumerate}
\item Author, Title, Journal, Year.
\end{enumerate}

\end{document}
```

## Required sections

Every generated review should include:

1. `Question`
2. `Overview`
3. `Main Findings`
4. `Gaps or Limitations`
5. `Bottom Line`
6. `References`

If the report includes images, also:

- store them under `literature_review_assets/images/`
- generate `literature_review_images.tex`
- `\input{literature_review_images.tex}` at the right place in the document body

## Escaping special characters

Escape these characters when they appear in normal text:

- `#` as `\#`
- `$` as `\$`
- `%` as `\%`
- `&` as `\&`
- `_` as `\_`
- `{` as `\{`
- `}` as `\}`

Handle backslashes and carets carefully:

- `\` can be represented as `\textbackslash{}`
- `^` can be represented as `\textasciicircum{}`
- `~` can be represented as `\textasciitilde{}`

## Citation formatting

This workflow does not require BibTeX. A simple numbered list in the `References` section is enough unless the user asks for a bibliography system.

Use a readable format such as:

```tex
\item Smith J, Doe A. Title of paper. Journal Name. 2024.
```

When including long URLs, prefer loading `xurl` and setting `breaklinks=true` in `hyperref` so links can wrap cleanly in the PDF instead of overflowing the page.

Do not mix a manual `enumerate` references list with a `thebibliography` environment in the same document unless there is a specific reason to do so. For this skill, a single `enumerate` list is usually the simplest and most reliable choice.

## Image manifest workflow

Only use this image workflow after explicitly asking the user whether the report should include images. If the user says yes, the agent should source them as part of the literature review workflow rather than waiting for the user to provide files. Prefer figures from cited papers or closely related explanatory sources, then create a manifest in the current working directory:

```json
[
  {
    "source": "https://example.com/figure1.png",
    "caption": "Conceptual overview of the intervention pipeline.",
    "credit": "Example source",
    "section": "Overview",
    "width": "0.85\\linewidth",
    "placement": "htbp",
    "label": "fig:overview"
  },
  {
    "source": "./notes/local-chart.png",
    "caption": "Outcome effects across included studies.",
    "section": "Main Findings"
  }
]
```

Then run:

```bash
uv run python .agents/skills/literature-reivew/prepare_report_images.py literature_review_images.json
```

The helper will:

- download or copy each image into `literature_review_assets/images/`
- reject unsupported formats such as SVG
- generate `literature_review_images.tex` containing `figure` environments
- update the manifest with the resolved `output_path` for each image

If no suitable figure can be sourced, omit images and say so explicitly in the review output.

Input the snippet where the images should appear:

```tex
\section{Overview}
...
\input{literature_review_images.tex}
```

Use packages suited to figures:

- `\usepackage{graphicx}` for `\includegraphics`
- `\usepackage{float}` only if exact placement such as `[H]` is required

## PDF conversion script

Run the helper from the current working directory:

```bash
uv run python .agents/skills/literature-reivew/render_latex_pdf.py literature_review.tex
```

Run the helper through `uv` so it uses the project-managed environment instead of the system Python.

The script looks for one of these TeX engines in order:

1. `tectonic`
2. `pdflatex`
3. `xelatex`

If none are installed, the script exits with an error and leaves the `.tex` file in place.

If `literature_review_images.json` exists, the renderer also checks that:

- `literature_review_images.tex` exists
- any prepared `output_path` assets listed in the manifest exist

This prevents compiling a report that references images before the image preparation step has been run.

## Troubleshooting

- If compilation fails, inspect the `.tex` file for unescaped special characters.
- If no PDF is created, check whether `tectonic`, `pdflatex`, or `xelatex` is available in the `uv` environment.
- If references contain unusual punctuation, simplify them before retrying compilation.
- If figures are missing, rerun `prepare_report_images.py` and verify the generated asset paths in `literature_review_images.json`.
